# C2 — Vonage Voice Linux SDK

Isolated test that verifies the **Vonage Voice Linux SDK** can join an existing Vonage voice call as a server-side WebRTC participant.

**Platform: Linux only** (the SDK ships as a native Linux binary). Use Docker on macOS.

> **Public Beta:** Vonage Voice Linux SDK integration is currently in beta.

## What is Vonage Voice Linux SDK?

**Vonage Voice Linux SDK** is a native Linux library that allows server-side applications to join Vonage voice calls as WebRTC participants. It provides:

- **WebRTC client capability:** Join a call as if you were a browser participant.
- **Media frame access:** Receive and send raw audio/video frames from/to the call.
- **Session integration:** Full participant lifecycle (join, publish, subscribe, leave).

In this project, the Voice Linux SDK is the **bridge between Vonage Video and Pipecat**. It joins the call (using credentials from C1), captures audio frames from participants, and feeds them into the Pipecat pipeline for processing.

## Purpose

This C2 test validates that:

- The Voice Linux SDK is properly installed and compatible.
- The SDK can authenticate to Vonage using the call credentials from C1.
- The SDK can establish a WebRTC connection and appear as a participant in the call.
- Media frames can flow from the call into a local application.

When complete, the SDK participant appears in voice test client, confirming call connectivity is working.

## Prerequisites

- Docker + Docker Compose (macOS) **or** Linux host with Python 3.13+ on Linux AMD64/ARM64
- Completed test **C1** — `VONAGE_CALL_ID` must be set in `.env`
- `VONAGE_APPLICATION_ID` and `VONAGE_PRIVATE_KEY` set in `.env`

### SDK versions (latest baseline)

This test tracks the latest stable Vonage SDK line and is currently validated with:

- `vonage>=4.8.0`
- `vonage-voice>=1.5.0`

If you already created a virtualenv, refresh to the newest compatible packages before running:

```bash
pip install --upgrade -r requirements.txt
```

## Setup (macOS — Docker)

```bash
# From the repo root
docker compose run --rm --build c2-voice-linux-sdk
```

## Setup (native Linux)

```bash
cd tests/c2_voice_linux_sdk

python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
# or with uv: uv venv && uv pip install -r requirements.txt

uv run python test_connector.py
# or without uv: source .venv/bin/activate && python3 test_connector.py
```

## What it tests

1. Generates a Vonage publisher token from existing call
2. Initialises the Voice Linux SDK with that token
3. Joins the call as a WebRTC participant (audio/video capable)
4. Stays connected for 5 seconds then disconnects cleanly

## Official Best Practices Applied

- Uses an official Vonage participant token generated by the Vonage Python SDK.
- Uses the current Vonage Python SDK token model (`TokenOptions`) for call tokens.
- Uses Linux runtime (Docker on macOS) because Voice Linux SDK is Linux-native.
- Keeps the test short-lived and disconnects cleanly to avoid dangling call participants.

## Troubleshooting

| Error                                      | Fix                                                         |
| ------------------------------------------ | ----------------------------------------------------------- |
| `OSError: libvoice_linux.so not found` | Must run on Linux — use Docker on macOS                     |
| `Connection refused`                       | Check `VONAGE_CALL_ID` is correct and the call exists |
| `Unauthorized`                             | Verify `VONAGE_APPLICATION_ID` and `VONAGE_PRIVATE_KEY`     |
